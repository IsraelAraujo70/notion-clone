# M4 API: search, public pages, and permanent deletion

M4 keeps workspace permissions at three roles:

- `owner`: reads, writes, manages members, publishes, and permanently deletes.
- `editor`: reads, writes, publishes, and permanently deletes.
- `viewer`: reads private workspace content only.

Every private route requires `Authorization: Bearer <session>`. A nonmember gets
`403`; a viewer gets `403` on writes. Public-page failures always return `404`
so a bearer cannot distinguish missing, revoked, trashed, or deleted content.

## `GET /search?q=&limit=`

Searches every workspace the authenticated user belongs to. `q` is trimmed and
must contain 2–200 characters. `limit` defaults to 20 and is capped at 50.

```json
[
  {
    "workspace_id": "…",
    "workspace_name": "Produto",
    "page_id": "…",
    "page_title": "Notas de lançamento",
    "page_icon": "🚀",
    "block_id": "…",
    "block_type": "paragraph",
    "snippet": "A busca nunca cruza workspaces.",
    "rank": 0.42
  }
]
```

The query joins `workspace_members` in SQL. A candidate is excluded when either
it or any ancestor is trashed. Titles carry more full-text weight than body text
and captions. The response contains plain text only; clients must not interpret
`snippet` as HTML.

## Public page links

Authenticated management routes:

- `GET /workspaces/{workspace_id}/pages/{page_id}/public-link`
- `POST /workspaces/{workspace_id}/pages/{page_id}/public-link`
- `DELETE /workspaces/{workspace_id}/pages/{page_id}/public-link`

`GET` and `POST` return the single active link:

```json
{
  "token": "…",
  "url": "https://reason.example/share/…",
  "created_at": "2026-07-10T12:00:00Z"
}
```

`POST` is idempotent while a link is active. `DELETE` returns `204`; a later
`POST` creates a new token. Owner and editor may manage links. Moving the page
or one of its ancestors to trash revokes affected links in the operation
transaction, and restore does not republish them.

`GET /public/pages/{token}` needs no session and returns:

```json
{
  "page": {
    "rootId": "…",
    "blocks": []
  }
}
```

The response contains only the published page and its non-page descendants.
Nested page blocks, their titles, presence, cursors, operation logs, and recent
editors are omitted. The endpoint is read-only and opens no WebSocket.

## `DELETE /workspaces/{workspace_id}/trash/{block_id}`

Permanently removes one root currently visible in the workspace trash. Owner
and editor may call it; a live block or a descendant hidden under another
trashed root is rejected.

```json
{
  "deleted_blocks": 4,
  "media_cleanup_queued": 1
}
```

The API returns `202` after the database transaction deletes the subtree and
queues its image keys. The worker deletes objects from S3 with retry. Operation
rows remain as the audit trail.

## Verification

`make eval-m4` creates owner/editor/viewer/nonmember fixtures against the local
stack and proves search isolation, role enforcement, anonymous publication,
subpage privacy, revocation, trash filtering, permanent deletion, and worker
media cleanup.
